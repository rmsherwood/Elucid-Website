Fresh Installation

The Quickstart Package consists on a complete Durpal! + Template + Various Extensions + Sample Content, excellent for beginner users to explore back-end settings and sample content. Installing the Quickstart is just like installing Drupal!, but you will have the full demo of the theme on your server. Once your environment is ready, follow this quick guide:

Localhost only: create a new folder in the htdocs folder for Xampp (www folder for Wamp). Example: mywebsite
Unpack all files and folders of the Template Package file and open the demo folder.

How to Fresh Installation

This instruction are for people that have experience manually installing Drupal and importing databases. If you do not feel comfortable performing these steps yourself, contact your server administrator for help. WARNING Do not import the provided SQL file into an existing Drupal database or you will lose all existing data.

Full Drupal installation comes without images, which was only for theme demonstration.

After purchasing theme, you will get a download package. Extract the archive somewhere on your computer.

step 1

Download a theme package of your choice (e.g. gavias_tico_full)

step 2

Extract package.

step 3

Goto folder fresh-installation/demo. Copy the entire contents of the demo folder to the location on your web server where you want your site to be accessed. If you want your primary domain to point to your install of Tico Theme, copy everything to the root folder. If you want your site to be accessed as a subsection of your domain (for example: yourdomain.com/drupal) then simply rename the demo folder to the desired name and upload to the root of your server..

step 4

Access your MySQL database and create a new DB (e.g. gavias_tico)

Import the SQL file fresh-installation/database.sql.zip to database for theme(e.g. gavias_tico). The PhpMyAdmin might be helpfull for managing your databases.

step 5

Install profile standard for drupal

step 6

Login to your site with the username admin and the password admin and visit admin/people to change the admin username and password again, visit admin/config/system/site-information to change the site-information again

Go through the Drupal configuration found in admin/config options and update all the necessary information to reflect your website. Also be sure to visit /admin/config/media/file-system and update the Files Directories.

And visit admin/config/development/performance Clear all cache drupal.

Finish

You're having a full website.

3. Save: Click save button to the block builder.

4. Assign Block: You need goto yourdomain.com/admin/structure/block and config assign block builder to region.(recommend: region before fullwidth)

Custom Post Type

Article

  • Body: The default body field.
  • Post Category: This is a term reference field that is linked to a vocabulary created specifically for this content type.
  • Post Image: The default Image field used for uploading the images.
  • Post Format: The display post format (Image, Gallery, Video, Audio).
  • Post Gallery: Gallery field used for uploading the images for Gallery post format.
  • Post Embed: Embed field user for Code video audio for Video, audio post format.

Gallery

  • Body: The default body field.
  • Gallery Thumbnail: The default Thumbnail field used for uploading the image thumbnail.
  • Gallery Images: Gallery field used for uploading the images for Gallery Slideshow.

Portfolio

  • Body: The default body field.
  • Image: The default Thumbnail field used for uploading the image thumbnail.
  • Portfolio Gallery: Gallery field used for uploading the images for portfolio.
  • Portfolio Embed:Embed field user for Code video.
  • Portfolio Tags:This is a term reference field that is linked to a vocabulary created specifically for this content type.

Testimonial

  • Body: The default body field.
  • Testimonial Image: The default Thumbnail field used for uploading the image thumbnail.
  • Testimonial Avatar: The Avatar field used for uploading the avatar.
  • Testimonial Embed: Embed field user for Code video.
  • Testimonial job:The job for testimonial.
  • Testimonial name:The name for testimonial.

Assigning block

Blocks are placed in regions via the Block Admin page Dashboard > Structure > Blocks . Here you can Enable, throttle, place, configure blocks and Add a region

Creating new a region

1. Open gavias_tico/gavias_tico.info.yml file (with a text editor)
2. Find copyright: Copyright
3. Insert code after: region_name = Region human readable name
4. Save gavias_tico.info.yml file and go to Configuration > Performance to clear site cache
